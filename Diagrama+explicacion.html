Sincronización en Tiempo Real Cómo se implementó

La sincronización se logra a través del Supabase JavaScript SDK, que se comunica con la base de datos PostgreSQL mediante llamadas a la REST API de Supabase. Cada operación CRUD (crear, leer, actualizar, eliminar) se ejecuta de forma asíncrona sin recargar la página:


1
El usuario agrega/edita/elimina una tarea en el navegador
2
La UI se actualiza  inmediatamente en memoria local (optimistic update)
3
app.js llama a  saveToSupabase() / updateInSupabase() / deleteFromSupabase()
4
Supabase guarda el cambio en PostgreSQL en la nube
5
Al iniciar sesión desde cualquier dispositivo, loadFromSupabase() recupera todas las tareas actualizadas
💡

¿Por qué Supabase? Justificación de selección

Supabase fue seleccionado por las siguientes razones:

✓ Gratuito para proyectos pequeños
✓ SDK JavaScript sencillo
✓ Auth integrado (email + OAuth)
✓ Base de datos PostgreSQL real
✓ Row Level Security (RLS)
✓ Sin necesidad de backend propio
✓ Dashboard visual para la DB
✓ Deploy en minutos
🔑

Login con Google y GitHub Flujo OAuth 2.0

El login social se implementó usando el protocolo OAuth 2.0, donde Supabase actúa como intermediario:


1
Usuario hace clic en  "Google" o  "GitHub"
2
signInWithOAuth() redirige al proveedor externo
3
El proveedor verifica la identidad y redirige a  Supabase /auth/v1/callback
4
Supabase crea/recupera el usuario y redirige a  la app en Vercel
5
onAuthStateChange() detecta la sesión y muestra la app
🛡️

Seguridad con Row Level Security (RLS) Aislamiento de datos por usuario

Supabase utiliza RLS (Row Level Security) de PostgreSQL para garantizar que cada usuario solo pueda ver y modificar sus propias tareas. Aunque dos usuarios compartan la misma base de datos, las políticas RLS filtran automáticamente los datos usando auth.uid() — el ID único del usuario autenticado. Esto significa que incluso si alguien obtuviera la anon key pública, no podría acceder a los datos de otro usuario.
